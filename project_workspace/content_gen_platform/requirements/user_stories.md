# 用户故事清单 — 内容生成平台

---

## 史诗 EP-01：用户与知识库管理

### US-001：用户注册与登录
**关联需求**：REQ-FUNC-001  
**作为** 一名内容创作者，**我希望** 能注册账号并登录系统，**以便** 拥有独立的内容创作空间。

**验收标准：**
- AC-001-01：Given 用户在注册页面填写了合法的用户名、邮箱、密码，When 点击注册，Then 系统创建账号、发送验证邮件，并跳转至登录页。
- AC-001-02：Given 用户提供已注册的邮箱和正确密码，When 点击登录，Then 系统返回 JWT Token，用户进入个人工作台。
- AC-001-03：Given 用户输入错误密码超过5次，When 继续尝试登录，Then 账号锁定15分钟，并提示用户。

### US-002：上传文档到知识库
**关联需求**：REQ-FUNC-002  
**作为** 一名内容创作者，**我希望** 能上传 PDF/Word/TXT/Markdown 文档到我的知识库，**以便** 在生成文案时使用这些文档作为背景知识。

**验收标准：**
- AC-002-01：Given 用户在知识库页面选择了一个合法格式的文档（≤50MB），When 点击上传，Then 文档出现在知识库列表，显示"处理中"状态。
- AC-002-02：Given 文档上传成功，When 系统完成向量化处理，Then 文档状态变为"可用"，用户收到通知。
- AC-002-03：Given 用户上传了不支持格式的文件（如 .exe），When 点击上传，Then 系统拒绝并提示支持的格式列表。
- AC-002-04：Given 用户上传文件超过50MB，When 点击上传，Then 系统拒绝并提示文件大小限制。

### US-003：检索与管理知识库文档
**关联需求**：REQ-FUNC-004  
**作为** 一名内容创作者，**我希望** 能检索、重命名和删除知识库中的文档，**以便** 保持知识库整洁有序。

**验收标准：**
- AC-003-01：Given 知识库中存在多个文档，When 用户输入关键词搜索，Then 系统返回名称或内容匹配的文档列表。
- AC-003-02：Given 用户点击删除某文档，When 确认删除弹窗，Then 文档从列表消失，对应向量索引同步清除。
- AC-003-03：Given 用户对文档点击重命名，When 输入新名称并确认，Then 文档显示新名称，内容不变。

## 史诗 EP-02：LLM 文案生成

### US-004：配置 LLM 提供商
**关联需求**：REQ-FUNC-005  
**作为** 一名内容创作者，**我希望** 能在设置页面配置 DeepSeek 或火山引擎 API Key，**以便** 系统使用我的 API 额度生成文案。

**验收标准：**
- AC-004-01：Given 用户在 LLM 设置页选择"DeepSeek"并输入 API Key，When 点击保存，Then 系统加密存储 Key，页面显示"已配置（DeepSeek）"及遮罩后的 Key。
- AC-004-02：Given 用户在 LLM 设置页选择"火山引擎"并输入 API Key，When 点击保存，Then 系统加密存储 Key，并允许用户指定模型名称（如 doubao-pro）。
- AC-004-03：Given 用户点击"测试连接"，When API Key 有效，Then 显示"连接成功"；Key 无效时显示错误信息。

### US-005：生成文案
**关联需求**：REQ-FUNC-006, REQ-FUNC-007, REQ-FUNC-008  
**作为** 一名内容创作者，**我希望** 能在工作台输入指令，选择平台类型和风格，系统自动检索知识库并生成文案，**以便** 快速获得符合平台调性的内容草稿。

**验收标准：**
- AC-005-01：Given 用户在工作台填写了生成指令、选择了平台类型（如小红书）和风格（如种草），When 点击"生成"，Then 文案以流式方式逐字显示在编辑区。
- AC-005-02：Given 用户开启了"使用知识库"选项，When 生成请求发送，Then 系统从知识库检索 Top-3 相关片段并附加到 LLM Prompt 的上下文中。
- AC-005-03：Given 文案正在生成中，When 用户点击"停止"，Then 生成立即停止，已生成的部分自动保存为草稿。
- AC-005-04：Given 文案生成完成，When 流式输出结束，Then 文案自动保存为草稿，显示字数统计。
- AC-005-05：Given 用户设置了字数限制（如300字），When 生成完成，Then 文案字数不超过设定限制（±10%允许误差）。

### US-006：确认并编辑文案
**关联需求**：REQ-FUNC-009  
**作为** 一名内容创作者，**我希望** 能对草稿文案进行编辑修改，然后标记为"已确认"，**以便** 确认后才允许发布或用于视频生成。

**验收标准：**
- AC-006-01：Given 用户查看草稿文案，When 在编辑区修改内容，Then 修改实时自动保存（防止数据丢失）。
- AC-006-02：Given 用户对文案满意，When 点击"确认文案"，Then 文案状态变为"已确认"，发布按钮和视频生成按钮变为可用。
- AC-006-03：Given 文案处于"已确认"状态，When 用户重新编辑内容，Then 文案自动回到"草稿"状态，需再次确认。

## 史诗 EP-03：多平台自媒体发布

### US-007：绑定自媒体平台账号
**关联需求**：REQ-FUNC-010  
**作为** 一名内容创作者，**我希望** 能在系统中绑定微博、小红书、微信公众号、微信视频号、今日头条等平台账号，**以便** 一键发布文案。

**验收标准：**
- AC-007-01：Given 用户在平台管理页点击"绑定微博"，When 完成 OAuth 授权流程，Then 微博账号出现在已绑定列表，显示账号昵称。
- AC-007-02：Given 用户在平台管理页点击"绑定小红书"，When 填写 API Key 并保存，Then 小红书账号出现在已绑定列表。
- AC-007-03：Given 用户解绑某平台，When 确认操作，Then 该平台账号从列表移除，相关 Token 安全删除。

### US-008：多平台一键发布
**关联需求**：REQ-FUNC-011, REQ-FUNC-012  
**作为** 一名内容创作者，**我希望** 能选择多个平台同时发布已确认的文案，**以便** 节省逐一发布的时间。

**验收标准：**
- AC-008-01：Given 用户在发布页面选择了3个目标平台，When 点击"立即发布"，Then 系统对每个平台异步发起发布请求，页面显示每个平台的发布状态（发布中）。
- AC-008-02：Given 系统向某平台发布成功，When 平台返回成功响应，Then 该平台状态更新为"成功"，显示发布链接（如平台返回）。
- AC-008-03：Given 系统向某平台发布失败，When 平台返回错误响应，Then 该平台状态更新为"失败"，显示错误原因，提供"重试"按钮。
- AC-008-04：Given 用户选择"定时发布"并设置时间（未来时间），When 点击"确认定时"，Then 系统保存发布任务，在指定时间自动执行发布。

### US-009：查看发布历史
**关联需求**：REQ-FUNC-013  
**作为** 一名内容创作者，**我希望** 能查看所有历史发布记录，**以便** 追踪每条内容的传播情况。

**验收标准：**
- AC-009-01：Given 用户进入发布历史页面，When 页面加载，Then 显示按时间倒序的发布记录，每条记录包含文案摘要、目标平台、发布状态、发布时间。
- AC-009-02：Given 发布记录显示"成功"，When 用户点击"查看"，Then 若平台返回了内容链接，跳转至对应平台页面。

## 史诗 EP-04：AI 视频生成

### US-010：生成视频分镜
**关联需求**：REQ-FUNC-014, REQ-FUNC-015, REQ-FUNC-019  
**作为** 一名内容创作者，**我希望** 系统能将已确认的文案自动拆解为结构化分镜提示词，**以便** 用于 AI 视频生成。

**验收标准：**
- AC-010-01：Given 用户在已确认文案页点击"生成视频"，When 系统处理完成，Then 显示分镜列表，每个分镜包含：序号、画面提示词、配音文案、时长、转场方式。
- AC-010-02：Given 分镜列表展示完成，When 用户检查第N和第N+1个分镜，Then 两个分镜的主体和场景描述保持逻辑连贯（不出现突兀跳跃）。
- AC-010-03：Given 用户对某个分镜的提示词不满意，When 手动修改画面描述或配音文案，Then 修改实时保存，不影响其他分镜。

### US-011：提交视频生成任务
**关联需求**：REQ-FUNC-016  
**作为** 一名内容创作者，**我希望** 确认分镜后，系统调用即梦 API 生成视频，**以便** 获得成片。

**验收标准：**
- AC-011-01：Given 用户已配置即梦 API Key，When 用户点击"开始生成视频"，Then 系统将所有分镜提示词提交至即梦 API，返回任务 ID，页面显示"视频生成中"进度条。
- AC-011-02：Given 即梦 API 任务完成，When 系统轮询到完成状态，Then 视频预览出现在页面，用户收到通知（站内消息）。
- AC-011-03：Given 即梦 API Key 未配置，When 用户点击"开始生成视频"，Then 系统提示"请先在设置页面配置即梦 API Key"，并提供快捷跳转链接。

### US-012：视频裁剪与导出
**关联需求**：REQ-FUNC-018  
**作为** 一名内容创作者，**我希望** 能对生成的视频进行分镜级别的裁剪与重排，**以便** 得到最终满意的视频。

**验收标准：**
- AC-012-01：Given 视频生成完成，When 用户进入裁剪页面，Then 时间轴上显示各分镜片段，可拖拽调整时长。
- AC-012-02：Given 用户在时间轴上删除某个分镜，When 点击删除按钮，Then 该分镜从时间轴移除，相邻分镜无缝连接。
- AC-012-03：Given 用户调整完裁剪方案，When 点击"导出视频"，Then 系统重新合成视频并提供下载链接（MP4格式）。

### US-013：配置即梦 API
**关联需求**：REQ-FUNC-016  
**作为** 一名内容创作者，**我希望** 能在设置页面配置即梦 API 的账号信息，**以便** 系统使用我的即梦账户生成视频。

**验收标准：**
- AC-013-01：Given 用户在"视频设置"页填写即梦 Access Key 和 Secret Key，When 点击保存，Then 系统加密存储，页面显示"已配置"及遮罩后的 Key 前4位。
- AC-013-02：Given 用户点击"测试即梦连接"，When API Key 有效，Then 显示"连接成功，账户余额：XXX"；Key 无效时显示具体错误信息。
