# 需求规格说明书 — 内容生成平台（Content Gen Platform）

---

## 1. 项目概述

**项目名称**：内容生成平台（Content Gen Platform）  
**技术栈**：Python + Django（后端）、Vue 3（前端）、Web Application  
**核心价值**：为内容创作者提供一站式解决方案——以私有知识库为上下文，调用国内大语言模型生成文案，一键分发至多个主流自媒体平台，并支持 AI 视频生成与裁剪。

---

## 2. 功能需求

### 模块一：用户与知识库管理

**REQ-FUNC-001**  
**来源锚定**：「为用户管理、存储自己的知识库（文档）作为上下文、知识」  
**描述**：系统支持用户注册、登录、个人空间隔离，每个用户拥有独立的知识库。

**REQ-FUNC-002**  
**来源锚定**：「为用户管理、存储自己的知识库（文档）作为上下文、知识」  
**描述**：用户可向知识库上传文档（支持格式：PDF、Word、TXT、Markdown），文档存储在用户私有空间，支持按名称、标签、上传日期检索。

**REQ-FUNC-003**  
**来源锚定**：「为用户管理、存储自己的知识库（文档）作为上下文、知识」  
**描述**：文档上传后，系统自动对文档进行向量化处理（Embedding），构建用户专属的语义检索索引，供 LLM 调用时作为上下文（RAG）。

**REQ-FUNC-004**  
**来源锚定**：「为用户管理、存储自己的知识库（文档）作为上下文、知识」  
**描述**：用户可查看、删除、重命名知识库中的文档；删除文档时同步清除对应的向量索引。

### 模块二：LLM 文案生成

**REQ-FUNC-005**  
**来源锚定**：「通过开放式API调用国内大语言模型（可选deepseek或者火山引擎）按照要求生成文案」  
**描述**：系统提供 LLM 提供商配置页面，用户可在 DeepSeek API 和火山引擎 API 之间切换，并安全存储对应的 API Key（加密存储）。

**REQ-FUNC-006**  
**来源锚定**：「按照要求生成文案」  
**描述**：用户在文案生成工作台可填写生成指令（Prompt），系统从知识库中检索相关文档片段作为上下文（Top-K 相关片段），拼装成完整请求发送给 LLM，返回生成的文案。

**REQ-FUNC-007**  
**来源锚定**：「按照要求生成文案」  
**描述**：文案生成支持配置参数：平台类型（微博/小红书/公众号/视频号/头条）、文案风格（专业/口语/幽默/种草）、字数限制、是否使用知识库上下文（可关闭）。

**REQ-FUNC-008**  
**来源锚定**：「按照要求生成文案」  
**描述**：生成的文案以流式方式逐步展示给用户（SSE/WebSocket），用户可在生成过程中停止。生成完成后，文案自动保存为草稿。

**REQ-FUNC-009**  
**来源锚定**：「文案确认后可以在自定义的自媒体发布」  
**描述**：用户可对已生成的文案进行编辑、修改后，标记为"已确认"状态，才能进入发布流程。

### 模块三：多平台自媒体发布

**REQ-FUNC-010**  
**来源锚定**：「文案确认后可以在自定义的自媒体发布，比如微博，小红书，微信公众号，微信视频号，头条等等发布」  
**描述**：系统支持以下平台的账号绑定与配置：微博、小红书、微信公众号、微信视频号、今日头条。每个平台以独立配置模块管理 OAuth 授权或 API Key。

**REQ-FUNC-011**  
**来源锚定**：「文案确认后可以在自定义的自媒体发布」  
**描述**：用户可从已确认的文案中选择目标发布平台（支持多平台同时勾选），一键触发发布。系统对每个目标平台异步发送发布请求。

**REQ-FUNC-012**  
**来源锚定**：「文案确认后可以在自定义的自媒体发布」  
**描述**：发布任务支持：立即发布 / 定时发布（指定日期时间）。发布状态实时更新（发布中/成功/失败），失败时显示错误详情。

**REQ-FUNC-013**  
**来源锚定**：「文案确认后可以在自定义的自媒体发布」  
**描述**：系统维护发布历史记录，用户可查看每条文案在各平台的发布状态、时间、链接（如平台返回）。

### 模块四：AI 视频生成

**REQ-FUNC-014**  
**来源锚定**：「视频生成，按照文案设计生词分镜提示词，提示词能够严格约束语音、视频」  
**描述**：用户可基于已确认的文案，触发视频生成流程。系统自动将文案拆解为多个分镜（Scene），每个分镜包含：画面描述提示词（Visual Prompt）、配音文案（Narration）、时长（秒）。

**REQ-FUNC-015**  
**来源锚定**：「提示词能够严格约束语音、视频」  
**描述**：每个分镜的提示词结构化设计，包含以下约束字段：
- `scene_prompt`：画面描述（主体、场景、风格、光线、镜头运动）
- `voice_style`：语音风格（语速、情感、音色）
- `duration_sec`：时长（2~10秒，步长1秒）
- `transition`：与前一分镜的转场方式（硬切/淡入淡出/推拉）

**REQ-FUNC-016**  
**来源锚定**：「视频生成调用即梦API，可以配置账号信息」  
**描述**：系统集成即梦（Jimeng）API，用户可在配置页面填写即梦账号的 Access Key 和 Secret Key（加密存储）。视频生成任务以异步方式提交到即梦 API，系统轮询任务状态并通知用户。

**REQ-FUNC-017**  
**来源锚定**：「视频的各个切片前后流畅，逻辑严谨」  
**描述**：分镜生成后，系统对分镜序列进行逻辑一致性校验：相邻分镜的主体、场景、叙事必须保持连贯，若不连贯则提示用户修改。最终视频由所有分镜按顺序合成。

**REQ-FUNC-018**  
**来源锚定**：「并且能够裁剪」  
**描述**：视频生成完成后，用户可在 Web 端进行简单裁剪操作：调整各分镜时长（拖拽时间轴）、删除某个分镜、调整分镜顺序，并重新合成导出最终视频。

**REQ-FUNC-019**  
**来源锚定**：「视频生成，按照文案设计生词分镜提示词」  
**描述**：分镜提示词支持用户手动编辑与 AI 自动生成两种方式；用户可在 AI 生成的基础上二次修改每个分镜的提示词参数。

## 3. 非功能需求

**REQ-NFUNC-001**  
**来源锚定**：「系统使用web application」  
**描述**：系统为 Web Application，前后端分离架构，前端 Vue 3，后端 Python + Django REST Framework，通过 RESTful API + WebSocket 通信。

**REQ-NFUNC-002**  
**来源锚定**：「可以配置账号信息」（多个 API Key/账号）  
**描述**：所有第三方服务凭证（LLM API Key、自媒体平台 OAuth Token、即梦 API Key）必须加密存储（AES-256），绝不以明文存储于数据库或日志中。

**REQ-NFUNC-003**  
**描述**：系统支持多用户并发使用，单用户的 LLM 生成请求响应时间 < 5秒（首Token），文案完整生成 < 60秒。[INFERRED — requires PM confirmation: 是否有明确的并发用户数要求？]

**REQ-NFUNC-004**  
**来源锚定**：「为用户管理、存储自己的知识库（文档）」  
**描述**：知识库文档单文件大小限制 50MB，用户总存储配额 2GB（可配置）。向量检索响应时间 < 1秒。[INFERRED — requires PM confirmation: 配额数值]

**REQ-NFUNC-005**  
**描述**：系统提供完整的操作审计日志，记录用户的关键操作（上传文档、生成文案、发布、视频生成）。

**REQ-NFUNC-006**  
**描述**：前端界面支持中文，响应式设计，支持现代浏览器（Chrome 90+、Firefox 90+、Safari 14+）。

## 4. 约束与边界

- **范围内**：知识库管理、文案生成、自媒体发布、视频生成与裁剪、用户账号管理。
- **范围外**：支付功能、内容审核（人工）、移动端 App、即梦 API 之外的其他视频生成服务（当前版本）。
- **平台 API 依赖**：微博/小红书/公众号/视频号/头条平台 API 须用户自行申请开发者权限；系统提供配置入口，但不保证各平台 API 的可用性和审核通过（属平台侧管控）。
